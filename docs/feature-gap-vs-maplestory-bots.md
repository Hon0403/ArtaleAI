# ArtaleAI 功能缺口對照（vs 常見楓之谷／Artale 輔助）

**狀態：v0.1** — 依市場常見外掛／腳本賣點與本專案現況盤點；作為後續 Console／Pipeline 優先序參考。  
**調查日期：** 2026-07-14

**對照產品範圍（公開行銷／社群常見說法，非逐一逆向）：**

- 經典服商業 bot 功能清單（技能輪轉、符文、換頻、日常等）
- Artale 圖色／視覺腳本常見賣點（自動刷怪、喝水、解輪、換頻）
- Artale 遊戲內建：寵物自動喝水、自動撿取、寵物自動加持

**本專案路線限制：** 電腦視覺 + 鍵鼠模擬；不以記憶體讀寫／注入為目標。商業 bot 的測謊破解、客戶端底層防偵測**不納入產品主軸**。

---

## 一、一句結論

接下來真正缺、且值得做的，多半已預留在主控台空殼：

1. 自動喝水  
2. 補助技能循環  
3. 攻擊技能輪轉  
4. 遇人換頻／Panic  
5. 符文等強制事件救援  

**撿錢／寵物補水／寵物自動加持**在 Artale 多半由遊戲本身覆蓋 → 外掛重複實作優先級低（可作無寵物備援）。

## 三、優先序（依掛機「會死／會卡住」）

### P0 — 掛機續航（主控台已畫空殼）

| # | 項目 | 既有資產 | 目標行為（白話） |
|---|------|----------|------------------|
| 1 | 自動喝水 | `PlayerVitalsSnapshot`、`groupBox7` | HP／MP 低於％時按補藥鍵 |
| 2 | 補助技能 | `groupBox9` | 依週期／略為隨機按 Buff 鍵 |
| 3 | 攻擊輪轉 | `groupBox8`（曾作休息；攻擊設定可另開或併入） | 主攻＋範圍技冷卻／機率，不永久只按同一鍵 |

**完成標準：** 非開發者在主控台勾選後，可「移動 + 打怪 + 不燒乾血魔 + 補助不掉」長掛。

### P1 — 異常不停機

| # | 項目 | 備註 |
|---|------|------|
| 4 | 遇人：暫停／小休／換頻 | 對齊小地圖 other players |
| 5 | 符文／卡死事件辨識與處理 | Artale 腳本痛點 |
| 6 | 藥水用完／死亡停損、可選回城 | 與 vitals、日誌聯動 |

### P2 — 體驗與擴充

| # | 項目 |
|---|------|
| 7 | SafeZones／RestrictedZones 編輯 UI |
| 8 | 跨圖巡邏流程 |
| 9 | Discord／系統通知「停了／死人／符文」 |
| 10 | `config.yaml` 記住勾選的怪物 |

### P3 — 低報酬或遊戲已覆蓋

- 純撿錢邏輯（寵物已撿）  
- 完整複刻寵物自動喝水／自動加持（無券使用者才有備援價值）  
- 對標商業 bot 全功能清單

## 四、Artale 特有取捨

```
經典服外掛賣點 ─┬─ Artale 寵物已覆蓋 → 降權（撿錢、補水、自動加持）
               ├─ Artale 圖色腳本痛點 → 升權（解輪、換頻、喝水備援）
               └─ 本專案優勢         → 路徑編輯 + CV 打怪閉環（維持）
```

產品定位應是：**視覺掛機閉環 + 技能節奏 + 異常停損**，而非「寵物功能第二套 UI」。

風險／報酬（夏普）直覺：

| 層級 | 項目 | 判斷 |
|------|------|------|
| 高夏普 | 喝水、Buff、技輪 | UI／資料半現成，投資小、存活率大 |
| 中夏普 | 換頻、符文 | 痛點真、實作量大 |
| 高尾部風險 | 測謊破解／注入防偵測 | 偏離 CV 路線，不納主軸 |

## 六、建議實作順序（合併條件）

```
P0-1 自動喝水（接 PlayerVitals + 顯示 groupBox7）
  → P0-2 補助技能（groupBox9）
  → P0-3 攻擊輪轉
  → P1 遇人／符文／停損
  → P2 體驗項
```

每階以「可手動驗收場景」為合併條件，避免一次做完商業 bot 全表。

## 結論

市場清單很長；**ArtaleAI 下一階不是變完整仿外掛，而是補齊掛機續航三件套（喝水／補助／技輪），再做異常不停機。**
